Nodejs是什么

👀 Node.js 不是一种语言，它是一个 JavaScript 运行环境。

Node.js 是用 C++ 和 JavaScript 开发的，用来在浏览器外运行 JavaScript 的运行时环境。

它让 JavaScript 也能在 服务器端 或 命令行里 执行，而不仅仅只能在浏览器里跑。

🧠 Node.js 是什么？

  • 是一个开源的、跨平台的 JavaScript 运行环境

  • 底层基于：

  • V8 引擎：Google Chrome 用的 JS 引擎，用 C++ 写的

  • libuv：一个跨平台的异步 I/O 库（也用 C 写的）

  • Node 自己封装了很多模块（比如 fs、http、path 等）

👇 Node.js 用什么写出来的？

组成部分 语言
V8 引擎 C++（谷歌的 JavaScript 引擎）
Node 核心 API C/C++（比如文件系统、网络模块）
内部工具 JavaScript（REPL、模块系统）

所以可以说：

💡 Node.js 是用 C++ + JS 写的运行时平台，专门用来让 JavaScript 能在浏览器之外运行。

✅ 用 Node.js 写代码时用什么语言？

💡 用的还是 JavaScript！（也可以用 TypeScript 等转译成 JS）

然后运行：

你写的是 JS，运行在 Node 的平台上。

所以，Node.js 不是一门语言，它是 JavaScript 的“服务器运行平台”，本身是用 C++ 和 JavaScript 写的，但你用它写代码还是写 JavaScript。

🧠 Node.js 是怎么让 JavaScript 在浏览器之外运行，还能读文件、连数据库、建服务器的！

🧩 Node.js 内部结构图

💡 各层详解：

层级 作用 开发语言
✅ 你写的 JavaScript 使用 Node API，比如 fs.readFile() JavaScript
✅ Node.js 核心模块 封装操作系统功能给 JS 用 C++ + JS
✅ libuv 管理事件循环、非阻塞 I/O C
✅ OS 层 实际读文件、开端口等 操作系统调用

🚀 举个例子：你写了一行 JS 读文件

执行流程是：

  1. 你调用 fs.readFile（Node 核心模块提供的接口）
  2. Node 内部通过 C++ 调用 libuv 发起异步读文件请求
  3. libuv 异步调用 OS 的 read() 系统调用
  4. 文件读完后，触发回调 → 再回到你 JS 代码中输出内容

💡 所以你写的是 JS，但幕后都是 Node 和 libuv 帮你“打下手”。

🧠 Node.js 为什么强大？

  • 借助 V8，它执行 JS 超快
  • 借助 libuv，它能做异步非阻塞 I/O（读写、网络都快）
  • 用 JavaScript 写全栈（前端 + 后端）也变得可能

Nodejs和vue的关系

Node.js 的两个角色

Node.js 可以干啥？ 举个例子
✅ 写后端（服务器） 接口、登录验证、数据库
✅ 做前端开发工具链 脚手架、打包器、开发服务器（vite/webpack）

👉 所以 Node.js 并不只是写后端，它其实是「运行 JS 的平台」，能跑任意 JS 脚本（不局限浏览器）

🎯 Vue 项目用 Node.js 干了什么？

Vue 项目虽然是前端的，但它的开发环境、构建工具、依赖管理全部都依赖于 Node.js 提供的生态工具链，而这些工具都通过 npm 来安装和管理。

打个比喻写 Vue 是“做饭”（前端页面），但你需要一些“厨具”（构建工具，比如 webpack、vite、eslint），这些厨具是基于 Node.js 的。

而 npm 就是“厨具商城”，负责给你送工具来。

虽然你写的是前端 Vue 项目，但你的开发环境其实是这样的：

任务 Node.js 在干啥
安装依赖 用 npm 下载第三方包
启动项目 用 vite / webpack 提供本地服务器
编译打包 把 .vue 文件打包成浏览器能识别的 HTML/CSS/JS
热更新 改一行代码浏览器自动刷新（HMR）也是 Node 服务提供的
脚手架 vue create 就是 Node 工具生成的项目结构

总结一句话：Vue 项目开发时的所有辅助工作，几乎都是由 Node.js 帮你完成的。

📦 Vue 依赖 Node.js 的点包括：

功能 依赖
vue-cli 脚手架 Node 环境
npm / yarn 安装依赖 Node 提供
构建工具（webpack、vite） 用 Node 写的
dev server 开发服务器 用 Node 起的本地服务

✅ 那什么时候 Node.js 才是“写后端”？

如果你用 Node.js + Express/Koa 写接口、数据库、登录注册等功能 —— 那就是写后端。

和你前端 Vue 项目不是一回事！

🔥 Vue 项目中“用到 Node.js”和“用 Node 写后端”是两回事！

场景 是否算后端开发？ 是否需要 Node.js
Vue 项目开发 + 构建 ❌ ✅
写接口、连接数据库 ✅ ✅
网页运行在浏览器 ❌ ❌（构建完 Node 可不参与）

你用 Vue 写前端页面时，是在用 Node.js 当“工具人”；

你用 Node.js 写接口时，才是在用它当“后端服务器”。

🔍 那 Vue 项目中到底用了哪些“Node 的东西”？

✅ 1. 项目脚手架（Vue CLI / Vite）

这个 CLI 就是 Node.js 写的工具，用 npm 安装。

✅ 2. 构建打包工具（Webpack / Vite / Rollup）

  • Vue 单文件组件（.vue）不能直接在浏览器跑，要用这些工具把它们打包成浏览器能识别的 HTML/CSS/JS。
  • 这些工具都基于 Node.js 写的，都是 npm 包。

✅ 3. 各种开发依赖

  • 比如：eslint、prettier、sass-loader、postcss、babel、typescript…
  • 这些都是 npm 包，不能在浏览器里直接用，必须 Node 运行

✅ 4. npm 也能装前端库（比如 Vue、axios）

这些最终会被打包进浏览器运行，但你依然用 npm 来统一管理它们版本和依赖。

虽然你的 Vue 项目最终运行在浏览器，但：

阶段 依赖 Node 吗？ 原因
开发阶段 ✅ 必须 使用 npm 安装工具、运行构建命令、调试等
构建阶段 ✅ 必须 用 webpack/vite 将项目打包成浏览器可运行的文件
运行阶段（浏览器） ❌ 不需要 最终只加载打包后的 JS、CSS、HTML

NPM

了解 npm 的原理和常用命令，对前端开发非常有帮助 💡

以下是简明易懂版，适合快速掌握和日常开发使用！

🧠 一、npm 是什么？

npm（Node Package Manager） 是 Node.js 的包管理工具，用来：

  • 下载 / 安装第三方包（比如 axios、vue、lodash）
  • 管理项目依赖
  • 执行项目脚本（如 npm run serve）

⚙️ 二、npm 的核心原理

原理 简述
📦 package.json 项目依赖说明文件，npm 安装时会按这个文件下载包
📁 node_modules 存放安装好的包
📄 package-lock.json 记录依赖包的确切版本，确保项目一致性
🌍 npm registry 包服务器地址（默认是 https://registry.npmjs.org）

✅ 安装依赖的流程大概是：

  1. 你运行 npm install
  2. 它读取 package.json
  3. 根据依赖树从 npm 官方仓库下载包
  4. 存到 node_modules 中
  5. 生成或更新 package-lock.json

📚 三、npm 常用命令大全（最实用）

📥 安装类

命令 说明
npm install 或 npm i 安装 package.json 中所有依赖
npm install xxx 安装某个包（默认加到 dependencies）
npm install xxx –save 显式加到 dependencies（生产依赖）
npm install xxx –save-dev 安装到 devDependencies（开发依赖）
npm install xxx@版本号 指定版本安装
npm install –force 强制安装依赖，忽略冲突
npm install –legacy-peer-deps 忽略 peerDependencies 冲突（常用于 React/Vue 项目）

🗑️ 卸载类

命令 说明
npm uninstall xxx 卸载包，并从依赖中移除
npm remove xxx 等同于 uninstall

📋 查看类

命令 说明
npm list 查看当前项目已安装的依赖树
npm list -g 查看全局安装的依赖
npm outdated 查看哪些依赖有新版本
npm view 包名 查看包的元信息（如版本、作者）

🚀 启动类（脚本执行）

在 package.json 中：

你可以执行：

命令 说明
npm run serve 运行 dev 服务器
npm run build 打包构建
npm run xxx 运行自定义脚本

🧹 清理类

命令 说明
rm -rf node_modules 删除依赖目录（Mac/Linux）
npm cache clean –force 清空 npm 缓存
npm rebuild 重新编译所有依赖

🌐 源管理类

命令 说明
npm config get registry 查看当前源
npm config set registry https://registry.npmmirror.com 设置为淘宝镜像（更快）
npm config delete registry 恢复默认源

🧠 常见 npm 问题排查技巧

问题 解决方式
command not found: vue-cli-service 缺少依赖 → npm install
Cannot find module ‘xxx’ 安装缺失包 → npm install xxx
包冲突或报错 尝试 npm install –legacy-peer-deps

✅ 总结：

npm 是前端开发的“包管理大管家”，会帮你管理依赖、版本、安装逻辑、脚本执行，配合 package.json 使用才是核心。

当项目出现依赖混乱、报错或版本冲突时，清理依赖并重新安装是非常常见且有效的做法。下面给你一套稳妥又彻底的清理+重装命令清单 👇

✅ 一键清理并重装依赖的标准流程

📦 Step 1：删除旧的依赖

这两项分别是：

  • node_modules：依赖实际安装的位置
  • package-lock.json：锁定的版本清单（不是必删，但推荐一起删）

🧹 Step 2：清空 npm 缓存（可选，但推荐）

用于清理本地缓存文件，防止缓存损坏影响安装

🔄 Step 3：重新安装依赖

npm 会重新读取 package.json，把依赖全部安装回来，并重新生成 package-lock.json。

🧠 如果你用的是

yarn或 pnpm

包管理器 清理命令 安装命令

pnpm pnpm install –force pnpm install

⚠️ 清理依赖常见适用场景

  • npm run serve 报依赖错误
  • 模块找不到（Cannot find module）
  • vue/webpack/react 插件报版本冲突
  • 项目长时间未维护，首次拉代码构建失败

yarn和npm对比

🧠 一、Yarn 是什么？

Yarn 是由 Facebook 开发的 JS 包管理工具，是 npm 的一个替代品，解决了 npm 早期的速度慢、版本不一致、并发差等问题。

它和 npm 的作用一样——安装/管理项目依赖包，但更加快速、安全。

⚙️ 二、Yarn 的工作原理（核心理解）

Yarn 通过以下几个机制提升效率和稳定性：

特性 说明
📦 离线缓存 安装过一次的包会缓存，下一次无需联网安装
🧩 确定性安装 使用 yarn.lock 锁定精确依赖，确保团队一致
⚡ 并行安装 多个依赖一起装，比 npm 串行快很多
✅ 一致性校验 通过 integrity 字段校验包完整性

🔧 三、Yarn 和 npm 的区别对照表

功能 npm yarn
安装命令 npm install yarn install
添加依赖 npm install xxx yarn add xxx
删除依赖 npm uninstall xxx yarn remove xxx
锁文件名 package-lock.json yarn.lock
缓存清理 npm cache clean –force yarn cache clean

yarn add 包名 -D 安装到 devDependencies（开发依赖）
yarn add 包名 -P 安装到 peerDependencies
yarn global add 包名 全局安装

🧠 五、项目什么时候用 Yarn 好？

  • 团队协作时：yarn.lock 能保证依赖版本一致
  • CI/CD 自动部署：更快更稳
  • 项目依赖多、装包慢：Yarn 并发性能好
  • 出现 npm 报错时，切换 yarn 可能避坑

✅ 总结：

Yarn 是更快、更可靠的 npm 替代方案，命令更清晰，安装更快，锁定更稳定。
